# A - Castle Renovation with Linked Doors

## 1. 問題概要

$N \times N$ マスの魔王城を舞台にしたパズル・最適化問題です 。
プレイヤー（魔王の部下）は、城内に**扉（最大$M$枚）**と**スイッチ**を自由に配置し、入口 $(0,0)$ から玉座 $(N-1,N-1)$ へ向かう勇者の到達をできるだけ長く足止め（最小行動回数を最大化）することを目指します 。
勇者は城の改造計画（配置）をすべて把握しており、玉座へ到達可能な場合は必ず**最小の行動回数**で到達します 。

### 制約・基本設定

* **盤面サイズ ($N$)**: 20固定


* **設置可能な最大扉数 ($M$)**: 50固定


* **スイッチの種類数 ($K$)**: 10固定（$0, 1, \dots, K-1$）


* **扉の型 ($2K$種類)**: $0, 1, \dots, 2K-1$


* *入口・玉座**: $(0,0)$ が入口、$(N-1,N-1)$ が玉座（いずれも初期状態は空きマス）



---

## 2. 詳細仕様とルール

### 扉とスイッチの連動

* **対応関係**: スイッチ $k$ （$0 \le k < K$）は、型 $2k$ の扉と型 $2k+1$ の扉を制御します 。


* **初期状態**:
* 偶数型（$2k$）の扉：**開いている**


* 奇数型（$2k+1$）の扉：**閉じている**




* **スイッチによる変化**: 勇者がスイッチ $k$ を押すたびに、対応する扉 $2k$ と $2k+1$ の開閉状態が反転します 。同じ型の扉はすべて同時に連動します 。



### 勇者の行動

勇者は1ターンに以下のいずれか1つの行動を取ります 。

1. **移動**: 上下左右に隣接する空きマスへ移動する（閉じている扉がある方向へは移動不可） 。


2. **スイッチを押す**: 現在いるマスにスイッチがある場合、それを押す（押さずに通り過ぎることも可能） 。



### 得点計算

* 勇者が有限回の行動で玉座に到達可能な場合、その最小行動回数を $T$ として、以下の得点が得られます 。



$$\text{得点} = \text{round}\left(10^6 \times \log_2 \frac{T}{N}\right)$$





* 勇者が玉座に**到達不能となった場合は不適正となり、得点は1点**となります 。



---

## 3. 入力フォーマット

標準入力から以下の形式で与えられます 。

```text
N M K
c_0
c_1
:
c_N-1

```

* $N$: 盤面サイズ（常に20）


* $M$: 設置可能な最大扉数（常に50）


* $K$: スイッチの種類数（常に10）


* $c_{i}$: 長さ $N$ の文字列。$j$ 番目の文字 $c_{i,j}$ は、マス $(i,j)$ が空きマスのとき `.`、障害物のとき `#` 。



---

## 4. 出力フォーマット

扉の配置、続いてスイッチの配置を標準出力へ出力してください 。

```text
D
d_0 i_0 j_0 g_0
d_1 i_1 j_1 g_1
:
d_D-1 i_D-1 j_D-1 g_D-1
S
p_0 q_0 s_0
p_1 q_1 s_1
:
p_S-1 q_S-1 s_S-1

```

### 扉の出力ルール

* $D$: 設置する扉の総数 ($0 \le D \le M$)


* $d_a$: 扉の向き


* `0`: マス $(i_a, j_a)$ と マス $(i_a+1, j_a)$ の間（上下隣接）


* `1`: マス $(i_a, j_a)$ と マス $(i_a, j_a+1)$ の間（左右隣接)




* $i_a, j_a$: 扉を配置するマスの座標


* $g_a$: 扉の型 ($0 \le g_a < 2K$)


* ※ 同じ2マスの間に複数の扉を設置することはできません 。



### スイッチの出力ルール

* $S$: 設置するスイッチの総数 ($0 \le S \le N^2$)


* $p_b, q_b$: スイッチを設置するマスの座標 $(p_b, q_b)$


* $s_b$: スイッチの種類 ($0 \le s_b < K$)


* ※ 同じマスに複数のスイッチを設置することはできません 。入口 $(0,0)$ や玉座 $(N-1,N-1)$ に設置することも可能です 。
